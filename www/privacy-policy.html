Privacy Policy

Effective: July 14, 2025

About VitalityCCM

This privacy policy applies to the VitalityCCM Chrome browser extension. If you have questions or concerns about this policy, you can contact the developer at contact@example.com.

Data We Access

When you run a check, the extension reads the text content and hyperlinks of your active browser tab solely to perform the checks you request (link checking and/or semantic checking). This data is not stored persistently and is not transmitted to any third party except as described in this policy.

Semantic Check & Data Transmission

When you invoke the semantic check feature, the page text extracted from your active tab is sent to the OpenRouter LLM API for analysis. This transmission occurs in one of two ways:

  • Via the developer's Vercel proxy — if you have not provided your own API key, requests are routed through the developer's Vercel-hosted proxy, which forwards them to OpenRouter.
  • Directly to OpenRouter — if you supply your own OpenRouter API key, requests are sent directly from your browser to the OpenRouter API without passing through the proxy.

In both cases, the page text is used solely to generate the semantic check result and is subject to OpenRouter's privacy policy.

API Key Storage

If you choose to provide your own OpenRouter API key, it is stored exclusively in chrome.storage.session. This storage is scoped to the current browser session and is automatically cleared when the browser session ends (e.g., when you close the browser). The API key is never written to disk, synced across devices, or transmitted to the developer.

No Personal Data Collected

VitalityCCM does not collect, store, or transmit to the developer any personally identifiable information (PII). This includes, but is not limited to:

  • Names or usernames
  • Email addresses
  • Browsing history
  • Location data
  • Any other information that could be used to identify you

Rate Limiting & IP Handling

The developer's Vercel proxy applies per-IP rate limiting to prevent abuse. Your client IP address is processed by the proxy solely for the purpose of enforcing this rate limit. The developer does not log, store, or otherwise retain IP addresses beyond what Vercel's infrastructure requires to route the request.
